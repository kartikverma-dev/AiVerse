# Project History

## Why it exists
To chronicle the origin and evolution of CredgeAiVerse from a simple terminology sheet to a living conceptual graph.

## When it was introduced
Initiated in 2025, with major refactorings in mid-2026.

## What problem it solved
It provided historical continuity, helping incoming engineers understand the transitions in database structure and API choices.

## Alternative ideas discussed
- Maintaining history in git commit logs only. Dismissed because commits lack architectural context and high-level reasoning.

## Why the final decision was chosen
A dedicated history file ensures clear, non-technical context is preserved for stakeholders and architects.

## Future possibilities
- Embedding interactive timelines into the history page showing previous versions of the schema and codebase.

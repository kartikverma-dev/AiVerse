# GitHub README Template

## Why it exists
To provide a public-facing introduction to the project for open-source contributors.

## When it was introduced
Maintained since 2025.

## What problem it solved
It onboarded developers quickly, explaining setup, architecture, and licensing.

## Alternative ideas discussed
- Keeping the repo private. Dismissed because community contributions are vital for cataloging AI terms.

## Why the final decision was chosen
A highly styled, comprehensive README is the best greeting for open-source developers.

## Future possibilities
- Interactive setup walk-through scripts.
